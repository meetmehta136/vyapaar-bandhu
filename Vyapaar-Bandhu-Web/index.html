How It Works

From Photo to Filed.
8 Steps, 0 Friction.

The complete journey from a business owner sending an invoice photo to a CA downloading a portal-ready GSTR-3B JSON.

Step 1

Business Owner Sends Invoice on WhatsApp

The owner takes a photo of any purchase invoice — Amazon, a local supplier, Flipkart — and sends it to the VyapaarBandhu WhatsApp number. Works on blurry, angled, handwritten, and printed invoices.

  • No app to install, no portal to learn
  • Works with any camera-quality photo
  • Instant acknowledgement: "Photo mil gayi! Processing…"
  • Handles printed, handwritten & photographed invoices
WhatsApp · VyapaarBandhu
Step 2

AI Extracts 9 GST Fields from the Image

NVIDIA Nemotron 12B VLM reads the image and extracts all critical GST fields. The GSTIN auto-correction system uses Modulo 36 checksum (the same algorithm the GSTN portal uses) to fix OCR errors.

  • Seller GSTIN, invoice number, date
  • Taxable amount, CGST, SGST, IGST, total
  • Product description for classification
  • GSTIN auto-corrects 7 common OCR errors: I/1, O/0, S/5…
OCR Extraction Result
Step 3

Maker-Checker → Classification → ITC Calculation

Extracted details are sent back to the user for confirmation. Once confirmed, a 3-layer AI pipeline classifies the invoice. The pure-Python compliance engine then calculates ITC eligibility without any AI — every rule is traceable to a specific GST Act clause.

  • Human-in-the-loop: user confirms or edits before saving
  • Layer 1: 400+ keyword rules (brands, product types)
  • Layer 2: BART zero-shot NLP for ambiguous descriptions
  • Layer 3: Fine-tuned IndicBERT on 1,995 Indian SME transactions
  • Section 17(5) blocked categories auto-flagged as ineligible
Classification & ITC Engine
Step 4

CA Reviews All Clients from One Dashboard

The CA's web dashboard shows all clients organised by compliance risk. Traffic-light status (green/yellow/red) tells the CA at a glance who is ready to file and who needs attention. One-click approve.

  • Green: all invoices in, ready to file
  • Yellow: some invoices missing, time remaining
  • Red: urgent — 3 days or fewer to deadline
  • CA can override any invoice category before approving
  • No manual entry, no spreadsheets, no back-and-forth calls
CA Dashboard · Client Overview
Step 5

PDF Export & GSTR-3B JSON — Portal Ready

CA downloads a filing summary PDF with their firm name on it — and a GSTR-3B JSON file in the exact format accepted by the GSTN portal. Upload directly. Job done. Automated reminders go out 7, 3, and 1 day before each deadline.

  • Filing summary PDF with CA firm's own branding
  • GSTR-3B JSON in GSTN-accepted format
  • White-label: business owners see the CA's name, not VyapaarBandhu
  • Automated deadline reminders: 7 days, 3 days, 1 day before
  • Admin panel: revenue metrics, all users, system health

Honest Assessment

Where It Can Go Wrong —
And How We Handle It

Every failure mode has a mitigation built into the system. Transparency is a trust-building feature for CAs.

👁️OCR Reads Invoice Incorrectly

AI vision model misreads a digit or letter. Happens more often on very blurry or handwritten invoices.

MitigationSystem never saves without human confirmation. Every field is shown back. Below 85% confidence → user must verify each field explicitly. CA approval is the second check.
🔢GSTIN Auto-Correction Guesses Wrong

Two different valid-looking GSTINs might both satisfy the Modulo 36 checksum in rare edge cases.

MitigationSystem tells user explicitly: "GSTIN corrected from X to Y — Valid (Maharashtra)". User can override with 'edit gstin' at any time.
🏷️Invoice Classification Error

A food invoice classified as Office Supplies leads to incorrect ITC eligibility — the most consequential error.

MitigationClassification shown to CA on dashboard. CA can override before approving. Vague descriptions ('miscellaneous') flagged for mandatory manual CA review.
⚠️Manipulated or Duplicate Invoice

Business owner sends inflated amounts or the same invoice twice to claim excess ITC.

MitigationDuplicate detection blocks same invoice number + GSTIN + user. Anomaly detection flags amounts 2.5× above the user's historical average for CA review.
⚖️GST Law Changes

GST rates change, new blocked categories added, or thresholds shift — making the compliance engine outdated.

MitigationCompliance engine is a single Python file with clearly labeled sections per GST Act clause. Any change is made in one place with a clear audit trail. No AI = no ambiguity.
📅Month-End Invoice Dump

Business owner sends 50 invoices on the 19th — defeats the purpose of running ITC tracking.

MitigationSystem handles any volume on any day. OCR API rate limits can be scaled. Reminder system designed to encourage early, consistent uploads throughout the month.
